Como Instalar DSK

Guia de instalação passo a passo para o ambiente local e produção.

Pré-requisitos

Ferramenta Versão mínima Verificar
Node.js 18.x ou superior node -v
npm / pnpm 9.x npm -v
Git 2.x git --version

Passo a passo

1

Clone o repositório

Faça o clone do projeto DSK para sua máquina local.

2

Instale as dependências

Execute o gerenciador de pacotes na raiz do projeto.

3

Configure as variáveis de ambiente

Copie o .env.example para .env e preencha os valores.

4

Inicie o servidor

Execute o comando de start e acesse localhost:3000.

BASH 
# 1. Clone o repositório
git clone https://github.com/b2u/dsk.git
cd dsk

# 2. Instale as dependências
npm install

# 3. Copie o arquivo de ambiente
cp .env.example .env

# 4. Inicie o servidor
npm run dev
ℹ️

O servidor iniciará na porta 3000 por padrão. Para alterar, defina PORT=XXXX no arquivo .env.

⚠️

Nunca compartilhe o arquivo .env nem o commite no repositório.

Teste: Login DSK

Testa o fluxo de autenticação com credenciais válidas e inválidas.

O que este teste cobre

✓

Login com credenciais válidas

Verifica redirecionamento para o dashboard após login correto.

✓

Mensagem de erro com credenciais inválidas

Garante que o sistema exibe feedback de erro adequado.

✓

Persistência de sessão

Valida que o token é salvo e a sessão permanece após reload.

Código do teste

TYPESCRIPT 
import { test, expect } from '@playwright/test';

test.describe('DSK — Autenticação', () => {

  test('login com credenciais válidas', async ({ page }) => {
    await page.goto('/login');
    await page.fill('[name=email]', 'admin@empresa.com');
    await page.fill('[name=password]', 'senha123');
    await page.click('button[type=submit]');
    await expect(page).toHaveURL('/dashboard');
  });

  test('exibe erro com credenciais inválidas', async ({ page }) => {
    await page.goto('/login');
    await page.fill('[name=email]', 'wrong@email.com');
    await page.fill('[name=password]', 'errado');
    await page.click('button[type=submit]');
    await expect(page.locator('.error-msg')).toBeVisible();
  });

});

Executar este teste

BASH 
npx playwright test tests/login.spec.ts --reporter=list

Teste: Fluxo Principal DSK

Cobre o fluxo completo de pedidos do início ao fim.

Cenários cobertos

1

Login e acesso ao painel

Autenticação e verificação do dashboard de pedidos.

2

Criação de novo pedido

Preenche formulário de pedido e valida criação.

3

Alteração de status

Atualiza o status do pedido e confirma mudança.

4

Exportação de histórico

Exporta relatório de pedidos e valida download.

TYPESCRIPT 
import { test, expect } from '@playwright/test';

test('fluxo principal DSK — pedido', async ({ page }) => {
  await page.goto('/login');
  await page.fill('[name=email]', process.env.TEST_USER);
  await page.fill('[name=password]', process.env.TEST_PASS);
  await page.click('button[type=submit]');

  await page.click('text=Novo Pedido');
  await page.fill('[name=descricao]', 'Pedido de Teste');
  await page.click('text=Confirmar');

  await expect(page.locator('text=Pedido de Teste'))
    .toBeVisible();
});

Teste: API Endpoints DSK

Verifica disponibilidade, contratos e respostas das rotas REST do DSK.

Endpoints testados

Método Rota Teste
GET /api/health Status 200 + body {ok: true}
POST /api/auth/login Retorna token JWT válido
GET /api/pedidos Array paginado no formato correto
POST /api/pedidos Cria e retorna novo pedido com ID
PATCH /api/pedidos/:id Atualiza status e retorna pedido atualizado
TYPESCRIPT 
import { test, expect } from '@playwright/test';

test('GET /api/pedidos retorna lista', async ({ request }) => {
  const res = await request.get('/api/pedidos');
  await expect(res).toBeOK();
  const body = await res.json();
  expect(Array.isArray(body.data)).toBe(true);
});
ℹ️

Os testes de API usam o APIRequestContext do Playwright, sem precisar abrir um browser.
